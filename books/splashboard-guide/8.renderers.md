fetcher が何を持ってくるかを決めるなら、 renderer はそれをどう描くかを決めます。 30 を超える種類が同梱されていて、ジャンルでざっと分けるとこうなります。

| カテゴリ | renderer |
| --- | --- |
| テキスト | `text_plain` / `text_ascii` ( blocks / figlet ) / `text_markdown` |
| チャート | `chart_sparkline` / `chart_bar` / `chart_line` / `chart_pie` / `chart_histogram` / `chart_scatter` |
| ゲージ | `gauge_line` / `gauge_circle` / `gauge_segment` / `gauge_battery` / `gauge_thermometer` |
| グリッド | `grid_heatmap` / `grid_calendar` / `grid_table` |
| リスト | `list_plain` / `list_cards` / `list_links` / `list_ranking` / `list_timeline` |
| バッジ | `status_badge` |
| アニメーション | `animated_boot` / `animated_typewriter` / `animated_scanlines` / `animated_wave` / `animated_splitflap` / `animated_postfx` / `animated_figlet_morph` |
| メディア | `media_image` / `loading` |

renderer は入力の shape ( Text / Bars / NumberSeries / Heatmap / Timeline / Ratio / Calendar / Badge / Entries / Image / PointSeries など ) で受け付けるものが決まっています。 fetcher が出す shape と合わない renderer を組み合わせると、 splashboard はその widget だけ shape 不一致のプレースホルダで描画します。組み合わせ一覧は reference matrix にあります:

https://splashboard.unhappychoice.com/reference/matrix/

この章では作者目線で 8 つを選び、 splash の見た目の中心を担うものを紹介します。

## 1. `text_ascii`

![同じ時計を figlet スタイルと blocks スタイルで描き分けた text_ascii の例](/images/splashboard-guide/demo-c8-textascii.png)

短いテキストを巨大に描く renderer です。

```toml
[[widget]]
id      = "clock"
fetcher = "clock"
format  = "%H:%M"
render  = { type = "text_ascii", style = "figlet", font = "banner", align = "center" }
```

スタイルが 2 系統あります。 `style` を省略すると `blocks` です。

- **`style = "blocks"`** ( デフォルト ): Unicode の半角ブロックで描く大文字。 `pixel_size` で `full` / `quadrant` / `sextant` を切り替えて密度を調整。省略時は枠の高さに合わせて自動選択
- **`style = "figlet"`**: クラシックな FIGlet の ASCII アート。 `font` は `standard` ( デフォルト ) / `small` / `big` / `banner` / `ansi_shadow` / `slant` / `isometric` / `doom` の 8 種類。未知の名前は `standard` にフォールバック

時計、レポ名、時間帯ラベル ( `morning` / `afternoon` ) など、 splash で大きく見せる要素はこの renderer で描けます。複数行のテキストには向かないので、複数行は `text_plain` を使います。

## 2. `chart_sparkline`

![chart_sparkline の bars スタイルと line スタイルの比較](/images/splashboard-guide/demo-sparkline.png)

`NumberSeries` をコンパクトな帯で描きます。直近 24 時間の CPU 、過去 1 週間の commit 数、今月の気温のような時系列を 1 行に収めるのに最適です。

```toml
[[widget]]
id      = "cpu"
fetcher = "system_monitor_cpu"
render  = { type = "chart_sparkline", style = "bars" }
```

スタイルが 2 つあります。

- **`style = "bars"`** ( デフォルト ): Unicode ブロックの縦棒。視認性が高い
- **`style = "line"`**: 1 行の Braille で描く線。さらに低背、細かい変動も拾える

枠に入りきらない値は古い方から切られ、直近の値が右端に揃うように描かれます。ゼロ値の列にも baseline が残るので、データの無と表示領域の余白が混ざりません。

## 3. `chart_bar`

![chart_bar の横棒スタイル ( 言語別ファイル数 )](/images/splashboard-guide/demo-chart-bar.png)

`Bars` shape のためのバーチャート。言語別ファイル数、コミット種別など、カテゴリと値のペアを描きます。

```toml
[[widget]]
id      = "lang"
fetcher = "github_languages"
render  = { type = "chart_bar", horizontal = true, max_bars = 6 }
```

デフォルトは縦棒 + ラベル下、 `horizontal = true` で横棒 + ラベル左に切り替わります。横長 / 縦長どちらのスロットでもバランス良く入るのが利点です。

カテゴリが多いときは `max_bars` で打ち切ります。

## 4. `grid_heatmap`

![grid_heatmap で 52 週ぶんのアクティビティを描いた状態 ( basic_heatmap で再現 )](/images/splashboard-guide/demo-c8-heatmap.png)

GitHub のコントリビューションカレンダー風の 2D グリッド。 `Heatmap` shape を 5 段階のヒート濃度で描きます。

```toml
[[widget]]
id      = "kusa"
fetcher = "github_contributions"
render  = "grid_heatmap"
```

グリッド幅がスロットを超える場合、右寄せで切られるので直近の列が必ず残ります。 1 年分の草を狭いスロットに置いても、最新が見える形になります。

合う fetcher は `github_contributions` / `git_commits_activity` / `git_blame_heatmap` / `basic_heatmap` ( TOML 直書き ) / `basic_read_store` ( JSON 出力 ) など。

## 5. `grid_calendar`

![grid_calendar: 今月のカレンダー ( 今日強調 ) 。 fetcher は calendar_holidays](/images/splashboard-guide/demo-grid-calendar.png)

`Calendar` shape を月表示で描きます。今日が強調表示され、イベント日 ( 祝日など ) には別の色が乗ります。

```toml
[[widget]]
id      = "cal"
fetcher = "calendar_holidays"
[widget.options]
country = "JP"
render = "grid_calendar"
```

`home_daily` の右側で使われています。 `clock_almanac` と組み合わせれば、今日が今年の何 % のような表示と並べることもできます。

## 6. `list_timeline`

![list_timeline で並ぶイベント ( 時刻 + タイトル + ステータス色 )](/images/splashboard-guide/demo-list-timeline.png)

`Timeline` shape ( = 時刻 + タイトル + ( 任意で ) 詳細 + ステータス ) を縦に並べます。左に時刻 ( `3h` / `2d` のような相対表記、または ISO 日付 ) 、 縦線で区切って、右にタイトル。

```toml
[[widget]]
id      = "prs"
fetcher = "github_my_prs"
render  = { type = "list_timeline", bullet = "●", max_items = 5 }
```

合うのは、 GitHub の通知、最近の commit 、最近のリリース、 RSS フィード、 hackernews / reddit の新着、など時系列 + ステータスのデータ全般です。

`list_cards` ( 各要素を panel で囲む ) や `list_ranking` ( 1, 2, 3 とランク番号を振る ) と相互に差し替えできるので、見た目を変えるだけで違う印象になります。

## 7. `gauge_circle`

![clock_ratio × gauge_circle で並ぶ年/月/週/日の進捗ゲージ](/images/splashboard-guide/demo-gauge-circle.png)

`Ratio` ( 0 - 1 の値 ) を塗りつぶしバーで描きます。 ratatui の Gauge をベースにした、密度の高い表示です。

```toml
[[widget]]
id      = "year"
fetcher = "clock_ratio"
[widget.options]
unit = "year"
render = { type = "gauge_circle", label_position = "center" }
```

`clock_ratio` の `year` / `month` / `week` / `day` と組み合わせると、今年 38 % 、今週 71 % のような時間の進捗ゲージが並びます。 `home_daily` の右側にある進捗群がそれです。

1 行で済ませたいなら `gauge_line` 、円形にしたいなら `gauge_segment` ( 名前と裏腹に円弧表現 ) があります。電池の絵が欲しいときは `gauge_battery` 、温度計が欲しければ `gauge_thermometer` 、と用途別に揃っています。

## 8. `status_badge`

![status_badge を ok / warn / error の 3 色で並べた状態](/images/splashboard-guide/demo-status-badge.png)

`Badge` shape ( = ステータス + ラベル ) を 1 個の色付きドット + テキストで描く renderer です。

```toml
[[widget]]
id      = "ci"
fetcher = "github_action_status"
render  = "status_badge"
```

色は status に応じて自動で切り替わります ( ok = 緑、 warn = 黄、 error = 赤 ) 。 `git_status` の Badge shape 、 `github_action_status` 、 `weather` の severe フラグなど、 「OK / 注意 / 異常」の 3 段階を狭いスペースで知らせたい場面で効きます。

## アニメーション系について

`animated_*` ( splitflap / typewriter / scanlines / wave / postfx / figlet_morph / boot の 7 種 ) は wrap 型 renderer で、 `inner` に本来描きたい renderer を渡すと初回描画時に演出が乗ります。動きは静止 PNG だと再現できないため本書では取り上げません。実物は `splashboard watch` で連続再生して確認するか、 [公式 showcase](https://splashboard.unhappychoice.com/showcases/) の GIF を見るのが手早いです。

シェル起動時の数百ミリ秒だけ動くギミックなので、デイリーで触っても飽きないバランスに調整されています。

## fetcher と組み合わせる前段として

ここまでで、何を取って、どう描くかの駒は揃いました。次の章では、その 2 つをどう組み合わせるか ( shape 一致のコツ、効く組み合わせ集 ) を扱います。
