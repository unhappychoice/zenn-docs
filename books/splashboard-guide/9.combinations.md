ここまでで何を取って、どう描くかの駒は揃いました。最後の一手は、その 2 つをマッチさせることです。

## shape ベースで考える

splashboard は fetcher と renderer を直接結ぶ仕組みではなく、間に **shape** という型を挟みます。

```
fetcher  ──>  shape  ──>  renderer
            (Text / Bars / Heatmap / Timeline / Ratio / Calendar / Badge / Entries / Image / NumberSeries / PointSeries)
```

組み立て可能 / 不可能はこの shape で決まります。

- 同じ fetcher が複数 shape に対応していることが多いです ( 例: `weather` は Entries / Text / PointSeries / NumberSeries / Bars / Badge の 6 つ ) 。 `shape = "..."` を `[widget.options]` で指定するか、 renderer 側で要求した shape が自動で選ばれます
- 1 つの renderer は特定の shape しか受け付けません ( 例: `grid_heatmap` は Heatmap のみ ) 。例外は `animated_*` で、こちらは全 shape を inner renderer 経由で受け付ける wrapper です

組み合わせがマッチしないと、その widget だけ shape 不一致のプレースホルダになります。 splash 全体は止まらないので安心です。

shape の早見表が [reference matrix](https://splashboard.unhappychoice.com/reference/matrix/) にあります。 fetcher の出力 shape と renderer の受け入れ shape が縦横に並んでいて、わからなくなったら見るのが早いです。

## まず覚えると応用が効く 10 組

作者が覚えておくと幅が広がると思う組み合わせを 10 個並べます。 TOML はどれも `[[widget]]` ブロックだけ載せます。 row はお好みで。

![home_daily preset: 巨大時計、進捗ゲージ、 calendar 、世界時計、 system info 、 sky panel など複数の組み合わせが同居している](/images/splashboard-guide/home_daily.png)

### 1. `clock` × `text_ascii`

```toml
[[widget]]
id      = "clock"
fetcher = "clock"
format  = "%H:%M"
render  = { type = "text_ascii", style = "blocks", pixel_size = "quadrant", align = "center" }
```

home の中央で安定して映える組み合わせです。

![clock × text_ascii](/images/splashboard-guide/demo-clock-figlet.png)

### 2. `git_status` × `status_badge`

```toml
[[widget]]
id      = "branch"
fetcher = "git_status"
render  = "status_badge"
```

project dashboard の左上に 1 個置くだけでも、 cd した瞬間のレポ間の切り替えが速くなります。

![git_status を 3 shape ( status_badge / text_plain / grid_table ) で並べた状態](/images/splashboard-guide/demo-git-status.png)

### 3. `git_commits_activity` × `grid_heatmap`

```toml
[[widget]]
id      = "commits"
fetcher = "git_commits_activity"
render  = "grid_heatmap"
```

GitHub 認証不要 ( ローカル `git log` 解析 ) なので、オフライン環境でも動きます。

![git_commits_activity × grid_heatmap: 直近 52 週の commit 草](/images/splashboard-guide/demo-git-commits-heatmap.png)

### 4. `github_contributions` × `grid_heatmap`

```toml
[[widget]]
id      = "kusa"
fetcher = "github_contributions"
render  = "grid_heatmap"
[widget.options]
login = "unhappychoice"
```

home の右半分にこれと直近 PR の timeline を並べる構成は鉄板です。

![Heatmap shape の例 ( ここでは git_commits_activity を使用、 github_contributions も同じ形 )](/images/splashboard-guide/demo-git-commits-heatmap.png)

### 5. `github_my_prs` × `list_timeline`

```toml
[[widget]]
id      = "my_prs"
fetcher = "github_my_prs"
render  = { type = "list_timeline", bullet = "●", max_items = 5 }
refresh_interval = 600
```

`list_cards` に切り替えれば panel で囲まれた印象に、 `list_ranking` にすれば番号付きランキング風になります。

![github_my_prs と同じ Timeline shape を basic_timeline で再現したサンプル](/images/splashboard-guide/demo-github-prs-shape.png)

### 6. `weather` × `chart_line`

```toml
[[widget]]
id      = "temp"
fetcher = "weather"
render  = "chart_line"
[widget.options]
latitude  = 35.6762
longitude = 139.6503
```

今日の天気を 1 グラフで把握する用途。 home の朝向け preset で効きます。 shape は renderer から自動推論されるので明示不要 ( `chart_line` が `PointSeries` を要求するので、 `weather` 側もそれを返します ) 。

![weather: 現在天気 + 24 時間気温の chart_line](/images/splashboard-guide/demo-weather.png)

### 7. `system_monitor_cpu` × `chart_sparkline`

```toml
[[widget]]
id      = "cpu"
fetcher = "system_monitor_cpu"
render  = { type = "chart_sparkline", style = "line" }
```

`style = "line"` の Braille 線は、 1 行のスペースに収まる動的な情報として優秀です。 home の上部に並べておくと、シェルを開いた瞬間にマシンの状態がわかります。

![chart_sparkline の bars / line スタイル ( basic_numbers で同等データを描画 )](/images/splashboard-guide/demo-sparkline.png)

### 8. `clock_ratio` × `gauge_circle`

```toml
[[widget]]
id      = "year"
fetcher = "clock_ratio"
render  = { type = "gauge_circle", label_position = "center" }
[widget.options]
unit = "year"
```

今年 38 % が一目でわかる表示です。 `unit = "year" / "month" / "week" / "day"` の 4 つを並べると、 `home_daily` の右側にある進捗群になります。

![clock_ratio × gauge_circle: 年 / 月 / 週 / 日 の 4 進捗ゲージ](/images/splashboard-guide/demo-gauge-circle.png)

### 9. `calendar_holidays` × `grid_calendar`

```toml
[[widget]]
id      = "cal"
fetcher = "calendar_holidays"
render  = "grid_calendar"
[widget.options]
country = "JP"
```

今月のカレンダーに今日と祝日が色付きで乗ります。 home の左下あたりに置いておくと、今日が何日で次の連休はいつかが一瞬で見えます。

![calendar_holidays × grid_calendar: 今月の日本祝日カレンダー](/images/splashboard-guide/demo-grid-calendar.png)

### 10. `git_repo_name` × `animated_splitflap` × `text_ascii`

```toml
[[widget]]
id      = "hero"
fetcher = "git_repo_name"
render  = { type = "animated_splitflap", inner = { type = "text_ascii", style = "figlet", font = "banner", align = "center", color = "panel_title" } }
```

per-repo dashboard のレポ名表示。シェル起動 / cd の初回だけパタパタめくれて、以降は止まったテキストとして残ります。 `splashboard watch` で観賞用にすると、ずっと動かせます。

![animated_splitflap × text_ascii: splash 名を映し終えた状態 ( アニメ自体は GIF で見るのが正確 )](/images/splashboard-guide/demo-splitflap.png)

## 避けたいパターン

shape が合わない以外にも、いくつか落とし穴があります。

### 重い fetcher を short refresh で並べる

`github_*` は API レート制限があるので、 `refresh_interval` を短くしすぎると 1 時間以内に枯渇することがあります。 splashboard はキャッシュ前提で動くので、 GitHub 系は 600 ( 10 分 ) から 1800 ( 30 分 ) で十分です。

### `animated_*` をたくさん同居させる

`animated_splitflap` や `animated_typewriter` を 5 個も 6 個も並べると、 splash がパチパチ動きすぎて落ち着きません。 1 つの splash の中で animated 系は 1 - 2 個までが経験則です。

### `text_ascii` で複数行を出す

`text_ascii` は短い 1 行に最適化されています。複数行を巨大に出したい場合は素直に `text_plain` を使うか、 1 行ずつ widget を分けます。

### 横長スロットに `chart_bar` の縦バー

カテゴリ数が多い `chart_bar` を横幅の狭いスロットに置くと、バー同士がくっついて読めなくなります。 `horizontal = true` で横バーに切り替えるか、 `max_bars` で上位だけ表示するのが現実的です。

## 自分の組み合わせを探す手順

新しい組み合わせを探すときの実用手順を置いておきます。

1. 何を表示したいかを日本語で 1 行書く ( 例: 最近 1 週間のコミット数の波形 )
2. その内容を出してくれそうな fetcher を [reference matrix](https://splashboard.unhappychoice.com/reference/matrix/) で名前検索 ( commit, sparkline など )
3. その fetcher が対応している shape の一覧を確認
4. その shape を受け付ける renderer から、見た目で選ぶ
5. `dashboard.toml` に 5 行書いて `splashboard` で確認
6. 期待と違ったら shape を切り替えて 4 に戻る
