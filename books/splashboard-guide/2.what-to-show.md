## ブラウザを開いたときと、ターミナルを開いたとき

ブラウザで GitHub のレポジトリページを開くと、 README 、最近の commit 、 Issue / PR 、 contribution heatmap などがトップに並んでいて、いま何が起きているかがぱっと見でわかります。

ターミナルを開いたときに見えるのは、プロンプト 1 行と、直前のコマンドの残骸くらいです。情報量の差は数十倍あります。

これは技術的な制約ではなく、ターミナルが次のコマンドを打つ場所という前提で設計されてきたためです。情報を映す場所として使われてこなかった、とも言えます。

コマンドラインに入ってからの数秒、まだ何も打っていない時間は、 GitHub のレポジトリトップを眺めるのと同じ用途に使えるはずです。 splashboard はそこを埋めるためのツールです。

## 既存のツールは何をカバーしているか

ターミナル起動時に何かを表示するツール自体はいくつかあります。役割が違うので、 splashboard との位置関係を整理しておきます。

### neofetch / fastfetch

https://github.com/fastfetch-cli/fastfetch

OS 、カーネル、 CPU 、メモリ、解像度などの **システム情報** を ASCII アートと並べて表示する系統です。

スクリーンショットで自分の環境を見せるのに向いています。レポジトリの状態のような **動的なデータ** は守備範囲外です。今日の天気も、レビュー待ちの PR も、コミット heatmap も出てきません。

splashboard は重なる部分もあります ( システム情報の widget も持っています ) 。向いている方向は、いまレポジトリで何が起きているかを映す側です。

### starship

https://starship.rs/

`starship` は 1 行のプロンプトに git ブランチ / 言語バージョン / CI 状態などを **集約する** ツールです。設定ファイルでカスタムできる柔軟さと表示の速さで、 dotfiles 文化の中でも特に普及しています。

ただし 1 行のスペースに集約するという設計上の制約があり、 ASCII アートも、 heatmap も、 timeline も、複数行の chart も入りません。 splashboard が目指す情報密度には届かないのが本質的な違いです。

splashboard と starship は競合しません。 splashboard で大きな表示を出し、入力中は starship で 1 行に集約する、という組み合わせが自然です。

### MOTD ( `/etc/motd` / `update-motd` )

Linux サーバに SSH で入ったときに表示される伝統的なテキストメッセージです。 `update-motd` を使えば動的にもできます。

ただし MOTD はサーバ管理者目線の機能で、ホストの uptime や利用規約を出すための仕組みです。レポジトリ単位の `cd` 連動も、リッチな TUI 描画も想定されていません。

### shell rc に書いた手製のスクリプト

`~/.zshrc` の末尾で `git status` や `gh pr list` を呼ぶ書き方も古くからあります。手早く始められますが、表示したい情報が増えるとすぐに管理しづらくなります。並べ替えるたびに rc を編集することになり、 dotfiles を別マシンに持ち越すのも面倒です。

splashboard はこの手製スクリプト的なアプローチを、 TOML での宣言的な記述と、 widget / レイアウトの抽象に乗せ替えたもの、とも言えます。

## splashboard が映そうとしているもの

GitHub のレポジトリページくらいの情報量を、ターミナルに出す。これが splashboard の目標です。

具体的にはこんなものを並べられます。

- **レポジトリの今**: branch 、 CI 状態、レビュー待ち PR 、最近の commit 、 contribution heatmap
- **今日の周辺**: 天気、月相、 RSS フィードの新着、カレンダーの予定、株価
- **集中したいときの情報密度**: sparkline 、 timeline 、 heatmap 、 chart などダッシュボード的な表示
- **見て楽しいだけのもの**: ASCII アート、 split-flap アニメ、 figlet フォントによる時計

これらを `dashboard.toml` に widget として並べると、シェル起動時と `cd` のたびに splash が出ます。

何を出すかは完全に好みで、情報を詰め込むのも、極限までシンプルにするのも自由です。次の章で実際に動かしてみます。インストール 2 コマンドと対話セットアップで、 5 分くらいで動く状態まで行きます。
