splashboard には 100 を超える fetcher が同梱されていて、いつ何をどこから取ってきてくれるかが、 fetcher の選択でほぼ決まります。

| カテゴリ | 主な fetcher |
| --- | --- |
| clock | `clock` / `clock_state` / `clock_almanac` / `clock_countdown` / `clock_ratio` / `clock_sunrise` / `clock_timezones` |
| system | `system_info_*` ( host / cpu / memory / kernel ... ) / `system_monitor_*` |
| git | `git_status` / `git_repo_name` / `git_commits_activity` / `git_recent_commits` / `git_latest_tag` / `git_churn` / `git_blame_heatmap` ほか |
| github | `github_my_prs` / `github_contributions` / `github_recent_releases` / `github_notifications` / `github_action_status` ほか |
| gitlab / linear | repo / issue / pipeline 系 |
| 外部 API | `weather` / `weather_forecast` / `calendar_holidays` |
| feed | `hackernews_*` / `reddit_*` / `lobsters_*` / `news_*` / `wikipedia_*` |
| ネタ系 | `random_quote` / `random_fortune` / `deal_*` ( セール情報 ) |
| 拡張 | `basic_read_store` ( 自作データを描く ) |
| 入力固定 | `basic_*` ( badge / bars / calendar / entries / numbers / points / timeline ... TOML に直書きするやつ ) |

全部の一覧と shape × renderer 互換表は公式 reference matrix にあります:

https://splashboard.unhappychoice.com/reference/matrix/

この章では、覚えておくと幅が広がる 8 つを選んで詳しめに紹介します。

## 1. `clock`

![clock × text_ascii (figlet)](/images/splashboard-guide/demo-clock-figlet.png)

シェルを開いた瞬間の体感に一番効くのが時計です。

```toml
[[widget]]
id      = "clock"
fetcher = "clock"
format  = "%H:%M"
render  = { type = "text_ascii", style = "figlet", font = "big", align = "center" }
```

`format` は strftime そのままです。 `%H:%M` で時刻だけ、 `%a %b %d` で `Wed May 17` のような表記になります。

splashboard には clock 系が 7 つあり、用途で選び分けます。

- **`clock`**: 単純な時計 ( 今の時刻 / 日付 )
- **`clock_state`**: 朝 / 昼 / 夜 のような時間帯ラベル ( `home_daily` の中央 figlet で使われている )
- **`clock_almanac`**: その日の年内位置、月相、曜日連番など暦周りのまとめ
- **`clock_countdown`**: 任意の日付までの残り日数 ( 誕生日、リリース日、勉強会など )
- **`clock_ratio`**: 年 / 月 / 週 / 日 の進捗率 ( `gauge_*` で出すと今年 38 % のような表示 )
- **`clock_sunrise`**: 日の出 / 日の入り ( 緯度経度を `[widget.options]` で渡す )
- **`clock_timezones`**: 複数タイムゾーンの今を一覧 ( リモートチームに )

時計まわりだけで splash の見た目がほぼ決まるので、まず触ると掴みやすいです。

## 2. `system_info_host`

![home_daily preset 内の system 情報セクション ( os / host / uptime / load / cpu / memory )](/images/splashboard-guide/home_daily.png)

`neofetch` 置き換えのキー fetcher です。

```toml
[[widget]]
id      = "host"
fetcher = "system_info_host"
[widget.options]
kind = "hostname"   # "terminal" / "os_name" / "os_version" / "hostname" / "shell" / "arch"

[[widget]]
id      = "os"
fetcher = "system_info_host"
[widget.options]
kind = "os_name"
```

1 つの widget が 1 つの情報を出す形なので、 OS / hostname / shell / arch のような複数行を出すなら、その分 widget を並べます。

合わせて使うことが多いのは:

- **`system_info_*`**: 静的な情報 ( bios / board / cpu / desktop / env / kernel / locale / machine / memory / timezone )
- **`system_monitor_*`**: 動的な情報 ( battery / cpu / host / load / memory / processes / uptime ) 、 sparkline / gauge と相性が良い

`system_monitor_cpu` を `chart_sparkline` で出すと直近の CPU 使用率の波形になり、 `system_monitor_battery` を `gauge_battery` で出すとバッテリーの絵そのものになります。

## 3. `git_status`

![project_github 内に出ている git_status ( branch + dirty 表記 )](/images/splashboard-guide/project_github.png)

現在のブランチ、 dirty / clean 、 upstream との ahead / behind を 1 つで取れます。

```toml
[[widget]]
id      = "branch"
fetcher = "git_status"
render  = "status_badge"          # Badge shape: clean = OK, dirty = warn
```

shape を切り替えると見た目が大きく変わります。

- `Entries` ( デフォルト ): `branch / ahead / modified / untracked` の key/value 表
- `Text`: `main  +2 −0  ◆3 ?1` のような 1 行
- `Badge`: clean / dirty の信号機 1 つ

project dashboard に何を 1 つだけ置くかと聞かれたら、 `git_status` の Badge 表示は最有力候補です。

## 4. `git_commits_activity`

![project_codebase 内の commit 草 ( 直近 52 週 )](/images/splashboard-guide/project_codebase.png)

GitHub のコントリビューションカレンダーのそのレポ版です。直近 52 週間を heatmap で出します。

```toml
[[widget]]
id      = "commits"
fetcher = "git_commits_activity"
render  = "grid_heatmap"
```

`project_github` preset に乗っている commit 草がこれです。 fetch は `git log` を解析するだけなので、 GitHub 認証不要 / オフラインで動きます。

ファイル単位でどこが燃えているかを見たくなったら、 `git_blame_heatmap` がほぼ同じ形のデータをファイル × 時間で出してくれます。

## 5. `github_my_prs`

![home_github 内の open PR timeline](/images/splashboard-guide/home_github.png)

開いている自分の PR を、レポを横断して取ってきます。レビュー待ち / 自分の作業中の PR を 1 画面で把握する用途です。

```toml
[[widget]]
id      = "my_prs"
fetcher = "github_my_prs"
render  = { type = "list_timeline", bullet = "●", max_items = 5 }
refresh_interval = 600
```

GitHub 系 fetcher は `GH_TOKEN` ( なければ `GITHUB_TOKEN` ) が要ります。一番楽な置き場所は `$HOME/.splashboard/secrets.toml` です。詳しくは Part 3 の secret 章で扱います。

近い fetcher:

- `github_assigned_issues`: 自分にアサインされた issue
- `github_notifications`: 受信中の通知 ( @ mention / レビューリクエストなど )
- `github_recent_releases`: フォロー対象 / 自レポのリリース

## 6. `github_contributions`

![home_github 内の contribution heatmap (草)](/images/splashboard-guide/home_github.png)

GitHub のプロフィールにある contribution calendar (草) を heatmap として出します。

```toml
[[widget]]
id      = "kusa"
fetcher = "github_contributions"
render  = "grid_heatmap"
[widget.options]
login = "unhappychoice"   # 省略すると認証ユーザー
```

`NumberSeries` shape も対応していて、週ごとに 1 つの値に潰すと sparkline で活動の波になります。

```toml
render = { type = "chart_sparkline", style = "bars" }
```

home dashboard で安定して効きます。

## 7. `weather`

![home_daily の中の weather 行 ( 現在天気 + 気温 )](/images/splashboard-guide/home_daily.png)

緯度経度を指定するだけで動きます。 API キー不要 ( Open-Meteo の無料 API ) なので、 splashboard を最初に触る人にとっての外部 API 体験として最適です。

```toml
[[widget]]
id      = "weather"
fetcher = "weather"
[widget.options]
latitude  = 35.6762
longitude = 139.6503
units     = "metric"      # "metric" / "imperial"
refresh_interval = 1800
```

shape の多さが特徴で、 1 つの fetcher を 5 通りで描けます。

| shape | 出るもの | 合う renderer |
| --- | --- | --- |
| `Entries` / `Text` | 現在の気温・天気 | `grid_table` / `text_plain` |
| `PointSeries` | 今後 24 時間の気温折れ線 | `chart_line` |
| `NumberSeries` / `Bars` | 今後 24 時間の降水量 | `chart_sparkline` / `chart_bar` |
| `Badge` | 大雨 / 雷 / 雪などの severe weather フラグ | `status_badge` |

ひとつ取ってきたデータをいくつもの見せ方で並べる、という splashboard らしさを最初に体感しやすい fetcher です。

## 8. `basic_read_store`

![basic_timeline / read-store 系の自作データを list_timeline で描いた例 ( 概念デモ )](/images/splashboard-guide/demo-list-timeline.png)

splashboard が標準で持っていないデータを映したい場合のエスケープハッチです。 `$HOME/.splashboard/store/<widget_id>.<ext>` ( `.text` / `.json` / `.toml` ) に好きな内容を書いておくと、 splashboard はそれを読んで指定 shape として描画します。

```toml
[[widget]]
id          = "habit"
fetcher     = "basic_read_store"
file_format = "json"
render      = "grid_heatmap"
```

そして `$HOME/.splashboard/store/habit.json` を cron / post-commit hook で更新します。これで習慣の達成度ヒートマップ、読書記録の timeline 、自分の任意の数字の sparkline が即席で組めます。

splashboard 本体を一切書き換えずに、任意のスクリプトの出力をそのまま splash に映せます。 Part 4 でもう少し深く扱います。

## 抜粋しなかったもの

抜けているもの ( gitlab / linear / 各種 feed / wikipedia / random_quote / deal / news ... ) も同じ流儀で、 `fetcher = "..."` と `[widget.options]` の組で動きます。出力 shape と説明が一覧されているので、気になる名前から飛ぶのが手早いです:

https://splashboard.unhappychoice.com/reference/matrix/

次の章では、これらの fetcher が出してくるデータをどう描くかの renderer 側の抜粋に入ります。
