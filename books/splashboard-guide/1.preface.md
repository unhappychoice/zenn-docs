## この本について

splashboard はシェル起動時と `cd` のたびにダッシュボードを描画する Rust 製の CLI ツールです。

https://github.com/unhappychoice/splashboard

公式ドキュメントが英語のみなので、日本語で順に読める資料として書きました。

https://splashboard.unhappychoice.com/

機能リファレンスではなく、 splashboard が **なぜそう動くのか** と **どう使うとどう映るのか** を一本の流れで追える読み物を目指します。網羅性は公式 docs の reference matrix に任せます。

https://splashboard.unhappychoice.com/reference/matrix/

## 想定読者

- ターミナルを毎日触っているが、起動時に出るのはプロンプト 1 行くらいで物足りないと感じている人
- dotfiles をいじるのが好きな人。 dotfiles に同梱できるツールを探している人
- `starship` 、 `neofetch` 、 `fastfetch` あたりの周辺ツールに興味がある人

Rust が書けなくても読めます。 fetcher / renderer を自作する話は最後の章で触り程度に出てきますが、それ以外は TOML を編集できれば十分です。

## 構成

全 4 部 14 章です。前半で作例を見ながら触り、後半で中で何が起きているかを掘ります。

- **Part 1: 触る・見る** ( 2 - 5 章 )
    - 既存ツールとの差分、 5 分で動かす、 home / project の作例ギャラリー、 3 種類の dashboard の使い分け
- **Part 2: 組む** ( 6 - 10 章 )
    - `dashboard.toml` の組み立て方、 fetcher / renderer の抜粋紹介、組み合わせのコツ、 read-store で凝った例
- **Part 3: 掘る** ( 11 - 13 章 )
    - データのパイプライン、シェル起動を止めない cache + daemon の設計、 trust モデルと secret 運用
- **Part 4: 広げる** ( 14 章 )
    - read-store で自作データを映す、自作 fetcher / renderer の触り

通して読むと、自分の dashboard を組めるところまで届くように構成してあります。

## まず触ってみたい人へ

ワンライナーでバイナリを入れます。

```bash
curl -fsSL https://raw.githubusercontent.com/unhappychoice/splashboard/main/install.sh | bash
```

対話セットアップを起動すると、シェル ( zsh / bash / fish / PowerShell ) を自動検出し、 home / project 用の preset とテーマをライブプレビューから選ぶだけで設定が書かれます。

```bash
splashboard install
```

新しいシェルを開けば、そこに splash が出ます。

詳しい話は次章から。
