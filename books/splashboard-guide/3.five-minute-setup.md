splashboard は、まず触ってもらうのが一番早いツールです。インストールから、新しいシェルを開いたら splash が出る状態まで、 5 分くらいで届きます。

この章では公式の対話セットアップを順に追います。

![splashboard 起動時の splash](/images/splashboard-guide/home_splash.png)

## バイナリを入れる

ワンライナーでバイナリを入れます。

```bash
curl -fsSL https://raw.githubusercontent.com/unhappychoice/splashboard/main/install.sh | bash
```

このスクリプトは Linux / macOS の x86_64 / aarch64 を自動判定し、最新の GitHub Release から該当バイナリを `~/.local/bin` に置きます。 Rust のツールチェインは要りません。

`~/.local/bin` 以外に置きたい場合は `INSTALL_DIR=...` で上書きできます。

別の手段も用意されています。

```bash
# cargo ( stable な Rust ツールチェインが要る )
cargo install splashboard

# Homebrew ( macOS / Linux )
brew install unhappychoice/tap/splashboard

# cargo-binstall ( prebuilt を GitHub Releases から )
cargo binstall splashboard

# Nix flake
nix profile install github:unhappychoice/splashboard
```

crates.io / GitHub Release は以下から確認できます。

https://crates.io/crates/splashboard

https://github.com/unhappychoice/splashboard/releases

Windows ( x86_64 ) は GitHub Release に prebuilt が上がっていて、 PowerShell から手動配置できます。

## 対話セットアップを起動する

バイナリが入ったら、対話セットアップを起動します。

```bash
splashboard install
```

シェル ( zsh / bash / fish / PowerShell ) を自動検出した上で、 4 つの選択画面を順に出します。すべて **ライブプレビュー付き** で、選んでいる preset / テーマがその場で右側に描画されます。確定する前に最終形が見える形です。

### 1. Home dashboard を選ぶ

新しいシェルを開いたときに出る splash です ( CWD が git レポ root でなければ ) 。 `cd` で出るのは `cd $HOME` のように $HOME 自体に降りたときだけで、それ以外の非 git ディレクトリへの `cd` では何も再描画されません。

| preset | 雰囲気 |
| --- | --- |
| `home_splash` | ASCII アート + 時計の signature 系 |
| `home_daily` | 天気 / カレンダー / RSS など朝に眺める情報 |
| `home_github` | contribution heatmap や受信中の通知 |
| `home_minimal` | 1 - 2 widget だけの最小構成 |
| `home_feed` | hackernews / reddit / lobsters などフィード中心 |

迷ったら `home_splash` で問題ありません。あとで乗り換えられます。

### 2. Project dashboard を選ぶ

git レポジトリのトップに `cd` したときに出る splash です ( `./.splashboard/` を同梱していないレポジトリの場合 ) 。

| preset | 雰囲気 |
| --- | --- |
| `project_splash` | レポ名の figlet + 概要系 |
| `project_github` | branch / CI / PR / commit など GitHub 寄りの今 |
| `project_minimal` | 必要最小限 |
| `project_codebase` | 言語比率や行数などコードベース寄り |

開発で常用するなら `project_github` が無難です。

### 3. テーマを選ぶ

`default` ( splashboard 専用の ocean パレット ) のほかに、定番テーマが揃っています。

- `tokyo_night`
- `catppuccin_mocha`
- `dracula`
- `nord`

### 4. 細かいデフォルト

トグルが 2 つあります。

- **背景塗りつぶし**: off にするとターミナル側の背景が透ける ( 透過ターミナルや light テーマで重要 ) 。デフォルトは on
- **最初の描画を fresh データまで待つか**: デフォルトは off 。 cached データから即描画し、新データはバックグラウンドで取り直すモードです。シェル起動を待たせない設計の核なので、特別な理由がなければ off のままで構いません

### 5. レビュー画面

最後にどのシェル、どの preset / テーマ、どのファイルが書かれるか、が並びます。

- Enter / `y` で書き込み
- Esc / `q` / `n` でキャンセル ( ファイルには一切触らない )

ここで戻れるので、確定前に確認できます。

## 何が書き換わるか

確定すると、以下が書かれます。

- `$HOME/.splashboard/home.dashboard.toml`: home dashboard 本体
- `$HOME/.splashboard/project.dashboard.toml`: project dashboard 本体
- `$HOME/.splashboard/settings.toml`: テーマ / トグルなど全体設定
- shell rc ( 例: `~/.zshrc` ) にマーカー付きの 1 行ブロック

rc に追記される行はこれだけです。

```bash
eval "$(splashboard init zsh)"
```

このマーカーブロックは再実行のときに上書きされるので、 rc に同じ行が何重にも増えることはありません。既存のファイルが書き換え対象になる場合も、 `.bak` サイドカーに退避してから書き込むので、いつでも 1 回の `mv` で戻せます。

## 動いているか確認する

新しいシェルを開きます。

```bash
$ zsh
```

選んだ preset の splash が出ます。これでセットアップは完了です。

![新しいシェルを開いたときの splash 表示例](/images/splashboard-guide/home_splash.png)

新シェルを開かずに今のシェルで確認したい場合、直接実行で再描画できます。

```bash
splashboard
```

git レポジトリのルートに `cd` すると、 home から project の splash に切り替わります。

## 出ないときの主な原因

splashboard は出すべきでない場面では意図的に描画しません。次のいずれかに当てはまっていると、何も描画されません。

- 標準出力がターミナルではない ( pipe / redirect / 非対話 )
- `TERM=dumb`
- `CI` / `SPLASHBOARD_SILENT` / `NO_SPLASHBOARD` のいずれかが set されている
- ターミナルが 40 × 16 文字より小さい

CI で意図的に止めているケースが多いので、ローカルで出ない場合は環境変数を確認するのが早いです。

GitHub 系 widget が出ているのにエラーが見える場合、 token がない可能性があります。一番早い対処は `secrets.toml` に書く方法です。

```toml
# $HOME/.splashboard/secrets.toml
GH_TOKEN = "ghp_..."
```

splashboard が起動時に環境変数として export するので、 rc を編集する必要はありません。詳しくは Part 3 の secret 章で扱います。
